# DAO Simulator Codebase Summary (2025-08-07)

High-level purpose
- A multi-agent simulation of a decentralized autonomous organization (DAO).
- Models members (agents) with different roles that create proposals, vote,
  fund projects, provide services, detect violations, trade tokens, and form guilds.

Core architecture & runtime
- Event-driven simulation where each timestep every agent executes a step() method.
- Agents inherit from a DAOMember base class that provides shared state and utilities.
- The DAO model holds global state: proposals, projects, treasury, token price,
  liquidity pools, event bus, guilds, disputes, services, and historical metrics.

Key domain objects
- Proposal: funding requests with lifecycle, votes, comments, funding and progress.
- Project: work item tied to a proposal with progress and duration.
- Dispute & Violation: dispute flow created by validators/auditors; violations
  impose reputation penalties.
- Treasury & LiquidityPool: manage funds and token swaps used by trader agents.
- Guild: persistent sub-DAO with its own treasury and projects.

Agent categories
- Governance: ProposalCreator, Delegator, Regulator, Auditor, Validator, Arbitrator.
- Economic: Investor, AdaptiveInvestor, Trader, RLTrader.
- Contributors & Services: Developer, ServiceProvider, BountyHunter.
- Specialized: Artist, Collector, Speculator, ExternalPartner, PlayerAgent.

Important flows
- Proposal lifecycle: create → fund/support → vote → approve → execute → complete.
- Monitoring & disputes: validators/auditors monitor projects, create disputes;
  arbitrators resolve disputes and apply violations.
- Reputation & incentives: reputation affects influence, work, and delegation;
  violations reduce reputation.
- Token economics: treasury, liquidity pools and traders drive token price dynamics.

Extension points
- Add new agent subclasses by implementing step() in new modules.
- Extend Proposal/Project models for new mechanics.
- Subscribe to the event bus for analytics or automated responses.

Observability & debugging tips
- Inspect the event bus for emitted events each timestep.
- Track proposal/project fields (current_funding, progress) to understand disputes.
- Monitor agent budgets (investment_budget, service_budget) and token price.
